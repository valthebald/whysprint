In public

Where should we do that? In public.

expose everyone to bits of the mentoring process:
mentor in public, in individual issues, in a public channel in irc, or in a room where people can overhear interactions between mentors and participants. seeing how mentors mentor is like training future mentors,
anyone who sees how mentors mentor will pick up parts of the mentoring habits.

Where do mentors come from?

Engage new contributors and experienced contributors

matching tasks, picking simple ones, answering questions we dont know and sharing how we find out is how mentors can help bring in people to participate,
and become repeat participants.
But where do the mentors come from?
This is one of the keys to mentoring in public.

Engage new contributors and experienced contributors
alike to participate in mentoring


Experienced contributors will see the mentor interactions and reuse those patterns themselves when helping people.
They can also see what mentors really do, and will see they can be a mentor. (Dont have to know answers to all the questions, only share how to find answers.)
Ask in advance for mentors, explain the "qualifications" (minimal qualifications and there will be support for mentors). Explain the expectations...
Ask in a pinch. If someone has done a task a few times, and a new person wants help doing it, ask that experienced person to help. Now they are a mentor.

Support mentors

instead of doing this for tasks the participants do, do it for the mentors

  • continue to be a resource
  • be available for questions
  • participate side by side
  • letting new people make mistakes (10 second rule)
  • offering specific suggestions to improve

Empower mentors

mentors will have ideas of how to improve mentor experience. let them.

  • document repetitive things mentors explain
    this makes a mentor memory so new mentors know what other mentors have figured out
    and can be shared directly with participants so they dont have to ask mentors
    Contributor Task Documents.

Ask mentors what they would change.

Those things can happen in the issue queues asynchronisly, real time, online for example in irc, or in person at events

Sprints

Sprints are usually in person focused events, with realtime interactions, lasting all day. Sometimes people participate remotely (drupal uses irc often for remote interactions).

Mentors at Sprints

small sprints, usually the organizer functions as a mentor.
Estimate the number of mentors: 1 for every 8 participants.

Sprint Preparation

Space needs. Conversation room for 6-10 people, schedule signup sheet outsite. Quiet room, experienced people, heads down working. New person area. Signs, direct people. Identify the tables.

Task lists. Tools: drupal has a special mentoring site used to track tasks (split up small parts of larger issues in our main issue tracker).
Working to change the issue tracker to enable that funcationality there.
google spreadsheet, especially for small events
Data: issue number, task, task difficulty level, skills needed, participant, mentor, status: wip, needs review, done


diverse set of task types, easy, med, difficult.
have tasks for those with a local dev env set up, those without.
document what tools people should have installed before, with references to other help docs how to set them up.
provide help getting them set up (let people know there will be help)

Wifi, outlet strips, Food. Coffee. tables.
